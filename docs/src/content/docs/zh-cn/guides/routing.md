---
title: 路由策略
description: CAB 内置路由策略与自定义路由规则。
---

CAB 决定每个网关请求由哪个模型和提供商处理。路由在网关层完成，随后才转发到上游。

## 解析顺序

1. **Agent 自动模式**——若 Agent 处于自动模式且绑定了策略，优先使用该策略。
2. **自定义路由规则**——路由页面中按 Agent User-Agent 模式匹配的规则。
3. **请求模型**——若客户端指定了目录中存在的模型 ID，直接使用。

使用 **路由 → 解释路由** 模拟请求，查看决策步骤和候选排序。

## 内置策略

可作为 Agent 策略或路由目标：

### Auto（自动智能选择）

解析请求文本，识别 coding / math / agentic / general 任务与复杂度；按 AA 能力指数加权评分；复杂度越高，最低能力门槛越高；最终以 **综合能力 / 有效成本** 排序，有效成本 = 输入价×3 + 输出价×1。

适合：混合工作负载，希望 CAB 按请求自适应。

### Balanced（平衡推荐）

按 **主能力指数 / (输入×3 + 输出×1)** 排序，在旗舰能力与真实 Agent token 比例之间取均衡。

适合：日常编程，兼顾成本。推荐作为默认策略。

### Intelligent（代码能力最强）

路由到已启用模型中 **AA coding index 最高** 的模型，同分则优先低价。

适合：高难度 Bug 修复、复杂重构、架构设计。

### Price（价格最惠）

按 token 总价（输入 + 输出）从低到高排序，过滤无效负价格。

适合：预算敏感场景和简单任务。

### Speed（速度优先）

路由到已启用模型中 **AA 实测输出速度最高**（`median_output_tokens_per_second`）的模型；同速则优先更低首 token 延迟（TTFT），再按有效成本排序。无 speed 数据的模型排在队尾；若全部无数据则降级为 **Price**。

适合：交互式编码、快速补全、对延迟敏感的工作流。

## 自定义路由规则

**路由** 页面可定义：

- **Agent 模式**——User-Agent 通配匹配（如 `codex`、`claude-code`、`pi`）
- **路由策略**——内置策略或指定模型
- **降级链**——主模型不可用时的备选

自定义规则会覆盖匹配 Agent 的默认解析。

## 降级

主模型或端点失败时，CAB 尝试备选候选（内置策略最多两个）。端点选择优先原生协议匹配，再回退到协议转换。

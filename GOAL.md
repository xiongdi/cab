# CAB启动后发生的事情

1. 下载最新model.dev数据并写入SQLite
2. 如果配置了key下载最新AA数据并写入SQLite，没有就用本地数据初始化

# 用户QuickStart

1. 打开提供商列表，搜索供应商
2. 填入key并开启
3. 打开模型列表，搜索模型并开启
4. 如果不使用Agent，这时可以直接访问http://localhost:3125
5. 如果使用CA，则打开CA列表进行设置
6. 选择自动模式就要选择一个策略，点击保持会写入到对应的CA的配置，这时候直接打开CA就能使用
7. 如果选择手动模式，这时候就会把模型列表写入对应CA的配置，打开CA后手动选择并使用

# 解决的痛点

1. 拥有多个提供商的订阅或者API
2. 人们往往不在乎使用的模型，知识希望在质量、速度、价格中取得一个平衡
3. 透明代理，任何发往LLM的数据都会被记录，防止CC这种不透明的CA留后面
4. 缓存命中，很多大模型的缓存命中价格非常低，提高缓存命中率能大幅降低费用，但是无量商家如CC，会在提示词里加时间戳这种来破坏缓存，于是CAB会对这些做优化

# 便捷的安装和使用体验

1. 提供cab-gui采用Tarui 2.0 + Svelte，支持多种操作系统和平台
2. 提供cab-srv后台常驻，开机启动
3. 提供cab-cli可供自己或者llm直接操控

4. 智能体

## V 模型总览

```
项目定义（左臂）                    确认（右臂）
─────────────────                  ─────────────────
需求分析 ─────────────────────────→ 用户验收测试
系统设计 ────────────────────────→ 系统测试
架构设计 ────────────────────────→ 集成测试
模组设计 ────────────────────────→ 单元测试
           ╲
            ╲  代码实现（V 底）
             ╲
```

维基原文要点：**左侧每一阶段，在右侧都有对应阶段**；且测试计划应在设计阶段早期制定，而非实现后才补写。

## 各阶段交付物（据维基条目）

### 需求分析（4 节）

1. 用户需要分析 — 面谈用户，建构理想系统
2. 用户需求文件（URD）
3. 功能需求整理 — 不决定软件设计方式
4. **用户验收测试计划** — 在此阶段早期制定

### 系统设计（5 节）

1. 软件规格说明书 — 开发蓝图
2. 系统架构与界面结构 — 菜单、架构
3. 业务场景与样例 — 视窗、报表
4. 实体图与数据字典
5. **系统测试计划**

### 架构设计（4 节）

维基条目未逐条列举小节，本站按架构阶段通行实践拆为：子系统划分、集成策略、接口契约、**集成测试计划**。

### 模组设计 / 低阶设计（5 节）

维基明确包括：模组拆解、逻辑/伪代码、数据库表、接口/相依性/错误/I/O、**单元测试计划**。

### 代码实现（4 节）

V 底编码活动；维基未展开小节，本站记录实现顺序、规范、构建与配置。

### 确认阶段（右臂）

| 阶段 | 维基目的 | 本站小节数 |
| --- | --- | --- |
| 单元测试 | 消除代码/单元级错误；隔离验证最小程式体 | 3 |
| 集成测试 | 验证子系统组装 | 3 |
| 系统测试 | 验证系统设计产出 | 4 |
| 用户验收测试 | 用户环境、实际数据、符合需求 | 5 |

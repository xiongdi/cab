# CAB Spec — 基于 V 模型的软件开发规范

基于 [Astro](https://astro.build) 的静态文档站，章节与小节结构对齐 [维基百科 V 模型（软件开发）](https://zh.wikipedia.org/wiki/V%E6%A8%A1%E5%9E%8B_(%E8%BB%9F%E9%AB%94%E9%96%8B%E7%99%BC)。

## V 模型章节对应

| 左侧（项目定义） | 右侧（确认） | 测试计划制定时点 |
| --- | --- | --- |
| 需求分析 | 验收测试 | 需求分析阶段制定 UAT 计划 |
| 系统设计 | 系统测试 | 系统设计阶段制定系统测试计划 |
| 架构设计 | 集成测试 | 架构设计阶段制定集成测试计划 |
| 模组设计 | 单元测试 | 模组设计阶段制定单元测试计划 |
| 代码实现 | — | V 模型底部 |

## 章与小节

### 前言
- V 模型概述、文档体系与追溯、术语、引用标准、角色职责

### 需求分析 → 验收测试
- 用户需要分析、用户需求文件（URD）、功能需求、**UAT 计划**、需求基线

### 系统设计 → 系统测试
- 软件规格说明书、架构与界面结构、业务场景与样例、实体图与数据字典、**系统测试计划**

### 架构设计 → 集成测试
- 架构视图、子系统划分、集成策略、接口契约、**集成测试计划**

### 模组设计 → 单元测试
- 模组拆解、逻辑与伪代码、数据库表、API/依赖/I/O、**单元测试计划**

### 代码实现
- 实现依据、编码规范、构建 CI、配置、代码审查

### 单元测试 / 集成测试 / 系统测试 / 验收测试
- 各章执行对应设计阶段已制定的测试计划，含隔离测试、真实数据 UAT 等 V 模型要点

### 附录
- V 模型对应关系表、模板清单、变更记录、参考资料、审批记录

## 开发

```bash
cd spec
npm install
npm run dev
```

## 重新生成文章骨架

```bash
node scripts/generate-docs.mjs
```

编辑 `src/data/chapters.ts` 调整导航，编辑 `scripts/generate-docs.mjs` 调整各小节正文模板。
